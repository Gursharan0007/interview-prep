Technical Questions

Show structured thinking — interviewers want to see your process, not just the answer

16
Technical
Walk me through troubleshooting a network printer that staff can't connect to.
+

I follow a layered approach — physical first, then network, then software.

Step 1 — Physical check: Is the printer powered on, showing "Ready" on the panel, and connected to the network port or Wi-Fi? Check indicator lights.

Step 2 — Network reachability: Find the printer's IP address from its control panel display. Ping it from the affected workstation. If ping fails — network issue. Check if the IP changed (DHCP lease expired and reassigned). If so, update the printer port on the workstation or assign a static IP to the printer.

Step 3 — If ping succeeds but printing fails: Check the print queue for stuck jobs, clear it, reinstall or update the driver, confirm the correct port is configured.

Step 4 — Isolation test: Try printing from a different workstation. If other machines work, the issue is user-specific. If nothing works, the issue is on the printer or server side.

I've done this regularly at both Mgadz and Delhi Public School.

Why this matters: "Troubleshoot network printers and copiers" is listed verbatim in the key accountabilities. Know this cold.

Technical
A classroom projector isn't displaying anything. How do you troubleshoot it?
+

Step 1 — Basic checks: Is the projector on? Wait 30–60 seconds for lamp warmup. Is the correct input source selected (HDMI 1 vs VGA vs wireless)?

Step 2 — Cable and connection: Inspect the cable connection at both ends. Swap in a known-good cable if available.

Step 3 — Laptop display settings: Press Windows + P to bring up display mode — toggle between Duplicate, Extend, and Projector Only. Sometimes the output is set to an inactive mode.

Step 4 — Resolution check: If the projector shows a signal but image is garbled or blank, check if the laptop resolution exceeds what the projector supports. Drop it to 1080p or lower.

Step 5 — Isolation: Test the projector with a different laptop. If it works, the issue is the original laptop's output settings or port. If still no signal, the projector itself may need service.

I handled this scenario regularly at Delhi Public School.

Why this matters: Classroom AV maintenance is a key accountability. AV troubleshooting will definitely come up — this structured answer shows you're methodical.

Technical
What's the difference between DNS and DHCP? When would each cause problems?
+

DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses to devices when they join a network. If DHCP fails, a device gets no IP address — or falls back to a self-assigned 169.254.x.x address (APIPA), which is a dead giveaway. The device won't be able to communicate on the network at all.

DNS (Domain Name System) translates domain names — like renison.uwaterloo.ca — into IP addresses so systems know where to route traffic. If DNS fails, the device may have a valid IP and can even ping IP addresses directly, but websites and named servers won't load. The classic symptom: "the internet is broken" but the network connection shows connected.

Quick diagnostic rule: Can't get on the network at all → suspect DHCP. Connected but nothing loads by name → suspect DNS.

Technical
A user says they can't connect to Wi-Fi. Walk me through your process.
+

Step 1 — Scope the problem: Is it just this user/device, or are others affected too? If multiple users are down, it's an infrastructure issue — escalate immediately.

Step 2 — Quick checks on the device: Is Wi-Fi enabled? Is Airplane Mode off? Is the correct SSID selected?

Step 3 — Forget and reconnect: Forget the network and reconnect fresh with the correct credentials. Sometimes cached bad credentials block connection.

Step 4 — IP address check: Run ipconfig. If they have a 169.254.x.x address, the device connected to the AP but didn't get a DHCP lease — possible DHCP issue or AP overload.

Step 5 — Driver check: Update or reinstall the wireless network adapter driver.

Step 6 — Test with another network: Try a phone hotspot. If it connects fine, the issue is specific to the corporate network (SSID, VLAN, credentials). If it won't connect to anything, hardware fault suspected.

Step 7 — Document and escalate if needed: If it's a network infrastructure issue, flag it to the appropriate team.

Technical
What's the difference between a virus, malware, and ransomware? How do you handle a suspected infection?
+

Malware is the umbrella term — any malicious software. Viruses are a type of malware that self-replicate by attaching to legitimate files. Ransomware encrypts a user's files and demands payment for the decryption key — it's currently one of the most damaging threats to organisations.

If I suspect an infection:
1. Isolate the device immediately — disconnect from the network (unplug ethernet, disable Wi-Fi) to prevent spread
2. Do not shut it down unless instructed — some malware destroys evidence on shutdown
3. Report to my manager immediately — this is not a solo fix
4. Preserve the state for investigation — avoid using the machine
5. Run a full scan with the organisation's endpoint security tool from a known-clean source
6. For ransomware specifically — check if backups are intact and unaffected before anything else

The most important thing is speed of isolation and communication — this is never something to handle quietly.

Technical
What is a VPN and when would a user need one?
+

A VPN (Virtual Private Network) creates an encrypted tunnel between a user's device and the organisation's network, making it appear as though the user is physically connected to the office network even when working remotely. Users typically need it to access resources that are restricted to the internal network — shared drives, internal applications, printers — when working from off-campus. Common helpdesk issues with VPNs include: the client not connecting (wrong credentials, expired certificate, blocked firewall port), slow speeds (server overload or split tunnelling misconfiguration), or resources not being accessible even when connected (routing or DNS issue on the VPN side). I've explained VPN use and basic troubleshooting to non-technical staff at Mgadz.

Technical
What is the OSI model? Can you walk through a few relevant layers?
+

The OSI model is a conceptual framework for understanding how data moves through a network — it has 7 layers, each responsible for a specific function.

For helpdesk work, the most practically useful layers are:

Layer 1 — Physical: The actual cables, ports, and hardware. If nothing works, start here — is the cable plugged in? Is the port lit up?

Layer 2 — Data Link: MAC addresses, switches. Relevant when diagnosing switch port issues or duplicate MAC conflicts.

Layer 3 — Network: IP addresses, routing. This is where DHCP and IP configuration troubleshooting lives — can you ping the gateway?

Layer 7 — Application: The software the user sees. Browser issues, app crashes, email client problems live here.

In practice, I troubleshoot bottom-up — confirm physical, then network, then application — which aligns with how the OSI model is intended to be used.

A faculty member's computer crashes mid-lecture. What do you do?

My first priority is minimising disruption to the class — not performing a full diagnostic.

After class, I'd investigate the root cause properly, document the incident, and either remediate the issue or replace the machine if it's a hardware fault. I'd also follow up with the faculty member to confirm everything was working before their next class.

The key principle: restore function first, investigate second.

Why this matters: Renison is described as patient and community-oriented. They want someone who is calm, discreet, and faculty-aware — not someone who creates more disruption than the problem itself.
